> CLI-утилита для управления прокси-серверами на базе Cloudflare Workers

## 📋 Описание

**Proxyflare** - современная CLI-утилита для создания и управления прокси-воркерами на платформе Cloudflare Workers. Проект представляет собой полную переработку существующего скрипта `flareprox.py` с применением лучших практик разработки на Python и Rust.

### Основные возможности

- **Мультиплатформенность**: Поддержка воркеров на TypeScript, Python, JavaScript и Rust (WASM).
- **Параллельный деплой**: Быстрое создание десятков воркеров одновременно.
- **Гибкая маршрутизация**: Обработка целей через query-параметры, заголовки или путь.
- **Локальный клиент**: Встроенные средства для тестирования и использования пула воркеров.
- **Управление артефактами**: Автоматическая сборка Rust-воркеров и копирование ресурсов.

## ⚖️ Сравнение с оригинальным flareprox

> [!NOTE]
> Проект **Proxyflare** вдохновлен утилитой [flareprox](https://github.com/MrTurvey/flareprox), но предлагает ряд существенных архитектурных улучшений:

| Особенность              | flareprox (Оригинал)                  | Proxyflare                                                                                                               |
| :----------------------- | :------------------------------------ | :----------------------------------------------------------------------------------------------------------------------- |
| **Языки воркеров**       | Только JavaScript                     | **TypeScript, JavaScript, Python, Rust (WASM)**                                                                          |
| **Архитектура**          | Монолитный Python-скрипт (~300 строк) | Модульное CLI-приложение (Typer, Rich, Pydantic)                                                                         |
| **Клиентская часть**     | Возвращает URL-адреса                 | Встроенный кастомный `httpx` Transport для автоматического прозрачного проксирования и ProxyManager для управления пулом |
| **Тестирование**         | Нет                                   | Полное Unit-покрытие и интеграционные тесты с `wrangler dev` + E2E                                                       |
| **Управление воркерами** | Создание синхронно                    | Параллельное создание, удаление, просмотр пула (пакетно)                                                                 |
| **Сборка артефактов**    | JS захардкожен внутри Python          | Выделены в отдельные файлы, Автоматическая динамическая сборка Rust через `cargo/worker-build`                           |

## 🏗️ Архитектура и алгоритм работы

### CLI Утилита
1. **Конфигурация**: Управление настройками Cloudflare через `.env` или системные переменные.
2. **Управление воркерами**:
    * `config verify`: Проверка токена и прав доступа API.
    * `config show`: Просмотр текущей конфигурации.
    * `create`: Параллельный деплой воркеров.
    * `list`: Список всех воркеров с префиксом `proxyflare`.
    * `delete`: Удаление воркеров по имени или маске.
    * `test`: Тестирование прокси-функционала через локальный клиент.

### Клиентская библиотека ([README](src/proxyflare/client/README.md))
1. **Manager**: Управление списком активных воркеров и выбор случайного узла.
2. **Transport**: Кастомный транспорт для `httpx`, прозрачно проксирующий запросы через воркеры.

### Воркеры (Прокси)
1. **Совместимость**: Очистка заголовков (Cloudflare-специфичные, Host) для корректного проксирования.
2. **CORS**: Встроенная поддержка CORS (`Access-Control-Allow-Origin: *`).
3. **Производительность**: Оптимизированный Rust воркер для высоконагруженных задач.

## 🛠️ Установка и использование

### Требования
- **Python 3.12+**
- **Rust (cargo)** (для компиляции WASM-артефактов, плагин `worker-build` устанавливается автоматически)
- **Node.js (npm/npx)** (для запуска локального тестирования через `wrangler dev`)

### Настройка Cloudflare API Token

Для работы **Proxyflare** требуется указать пользовательский Cloudflare API токен. Создайте Custom Token в панели управления Cloudflare (`My Profile -> API Tokens -> Create Token -> Custom Token`) со следующими **четырьмя** обязательными разрешениями:

1. **User** -> **API Tokens** -> **Read**
2. **Account** -> **Workers Scripts** -> **Edit**
3. **Account** -> **Account Settings** -> **Read**
4. **Zone** -> **Workers Routes** -> **Edit**

> [!WARNING]
> Не ограничивайте токен конкретными зонами в разделах "Zone Resources" или "Account Resources". Оставьте значения **Include -> All accounts** и **Include -> All zones**, иначе эндпоинт верификации (`verify`) будет возвращать ошибку `1000 Invalid API Token`.

Затем укажите выпущенный токен и ID вашего аккаунта в переменных окружения (например, в файле `.env`):
```bash
PROXYFLARE_API_TOKEN="ваш_custom_токен"
PROXYFLARE_ACCOUNT_ID="ваш_account_id"
```

### Установка

Самый простой способ установить Proxyflare — через pip (или uv pip):

```bash
pip install proxyflare
```

После установки вы можете проверить корректность настройки с помощью встроенной команды:
```bash
proxyflare config verify
```

### Сборка из исходников (для разработки)

Проект использует кастомный билд-хук `hatchling`, который автоматически компилирует Rust-воркеры при установке пакета. Рекомендуется использовать `uv` для быстрого управления зависимостями и сборки утилиты:

```bash
# Установка зависимостей проекта
uv sync

# Установка как системной утилиты
uv tool install .

# Установка пакета в режиме разработки (запустит сборку Rust-воркера)
uv pip install -e .
```

### Разработка и тестирование

1. **Unit-тесты** (Быстрые, локальные):
```bash
uv run pytest tests/unit
```

2. **Интеграционные тесты** (Локальный wrangler):
```bash
uv run pytest tests/integration
```

3. **E2E тесты** (Remote Cloudflare):
```bash
uv run pytest tests/remote/test_e2e.py
```
